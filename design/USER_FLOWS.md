# Velvet Hour Audit - User Flows

## 1. Landing Page to Audit Start
1.  **Visitor lands** on Landing Page.
2.  **Hero Section:** Strong value prop ("Agency-level insights for small businesses, in an hour").
3.  **Input Field:** "Enter your website URL" + "Get Free Audit" button.
4.  **Submission:** User enters URL and clicks button.
5.  **Scan Initiation:** Redirect to the Scan Progress page.

## 2. Audit Scan (The "Velvet Hour")
1.  **Loading Screen:** Elegant animation (e.g., a stylized hourglass).
2.  **Category Checklist:** 17 categories are listed. As the AI scans, they check off or show progress:
    - SEO
    - Performance/Speed
    - Accessibility
    - Mobile Responsiveness
    - Security
    - Content Quality
    - Conversion Rate Optimization (CRO)
    - Social Media Integration
    - Local Business Presence
    - Technical Health
    - UI/UX Best Practices
    - Branding Consistency
    - Customer Trust Signals
    - Lead Generation
    - Analytics Setup
    - Legal/Compliance (GDPR, etc.)
    - Competitive Benchmarking
3.  **Completion:** Once all categories are scanned, "View My Report" button appears.

## 3. Audit Report View
1.  **Executive Summary:** Overall Velvet Score (0-100), high-level breakdown of "Quick Wins" vs "Strategic Fixes".
2.  **Category Cards:** Each of the 17 categories shows a score and a summary.
3.  **Issue Detail:** Clicking a category expands to show specific issues:
    - Issue description.
    - Business impact (High/Med/Low).
    - Effort to fix.
    - The "Fix" (what needs to be done).
4.  **Fix Selection:** A "Fix it for me" button next to actionable issues.

## 4. Fix Purchase & Implementation
1.  **Cart/Selection:** User selects issues they want fixed.
2.  **Pricing:** Transparent flat-fee pricing shown.
3.  **Checkout:** Secure payment (Stripe integration).
4.  **Post-Purchase:** User is prompted to provide necessary access (e.g., WordPress admin, Shopify staff account) via a secure form.

## 5. Customer Dashboard
1.  **Overview:** Total score improvement over time.
2.  **Active Fixes:** Status of purchased fixes (Pending Access, In Progress, Review, Completed).
3.  **Audit History:** Access previous reports.
4.  **Re-audit:** Trigger a new scan to see updated scores after fixes are applied.

## 6. Admin Dashboard (Internal)
1.  **Orders:** List of purchased fixes.
2.  **Customer Management:** View customer sites and reports.
3.  **Fix Fulfillment:** Workflow for the team to track implementation progress.
